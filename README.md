# nproject.site — Спортивный секундомер

Проект представляет собой спортивный секундомер с интегрированным виджетом погоды. Развёрнут как коммерческий продукт с полностью автоматизированным GitOps-пайплайном.

## Описание

Идея простая: перед пробежкой спортсмену важно знать температуру и скорость ветра, а во время тренировки — точно засекать круги.

Фронтенд написан на чистом HTML/CSS/JS, бэкенд — на FastAPI. Мы сознательно отказались от тяжёлых фреймворков, чтобы Docker-образ оставался лёгким, а сборка проходила максимально быстро.

**Основная ценность проекта** — в автоматизации доставки.

## Архитектура

Архитектура построена по классическому GitOps-подходу:

- **Исходный код** приложения хранится в репозитории `project-backend`
- **Манифесты Kubernetes** — в отдельном `k8s-manifests`
- **При пуше** в ветку `main` GitHub отправляет вебхук, который инициирует сборку в Jenkins
- **Jenkins** собирает Docker-образ, публикует его в приватный Registry и автоматически обновляет тег образа в репозитории с манифестами
- **ArgoCD** отслеживает изменения в Git и применяет их в кластере
- В случае проблем доступен мгновенный откат через интерфейс Argo или revert-коммит

## Стек технологий

- **Язык:** Python 3.11
- **Фреймворк:** FastAPI
- **Контейнеризация:** Docker, приватный Registry (Yandex Cloud)
- **Оркестрация:** Kubernetes (Yandex Cloud)
- **CI/CD:** Jenkins (CI), ArgoCD (CD)
- **Мониторинг:** Prometheus + Grafana (метрики), Loki (логи)
- **Сеть:** Nginx Ingress, cert-manager (HTTPS)

## Безопасность

- **Секреты** не хранятся в Git в открытом виде — используется Sealed Secrets
- **Данные в etcd** зашифрованы на уровне хранилища (Encryption at Rest)
- **Бэкапы etcd** выполняются по расписанию, ключи шифрования хранятся изолированно
- **Rollback** выполняется через ArgoCD или откатом коммитов в репозитории манифестов
