## О проекте

Проект представляет собой спортивный секундомер с интегрированным виджетом погоды. Развёрнут как коммерческий продукт с полностью автоматизированным GitOps-пайплайном.

Идея простая: перед пробежкой спортсмену важно знать температуру и скорость ветра, а во время тренировки — точно засекать круги. Фронтенд написан на чистом HTML/CSS/JS, бэкенд — на FastAPI. Мы сознательно отказались от тяжёлых фреймворков, чтобы Docker-образ оставался лёгким, а сборка проходила максимально быстро.

## Как устроен деплой

Основная ценность проекта — в автоматизации доставки. Архитектура построена по классическому GitOps-подходу:
- Исходный код приложения хранится в репозитории `nproject-backend`, манифесты Kubernetes — в отдельном `k8s-manifests`.
- При пуше в ветку `main` GitHub отправляет вебхук, который инициирует сборку в Jenkins.
- Jenkins собирает Docker-образ, публикует его в приватный Registry и автоматически обновляет тег образа в репозитории с манифестами.
- ArgoCD отслеживает изменения в Git и применяет их в кластере. В случае проблем доступен мгновенный откат через интерфейс Argo или revert-коммит.

## Стек

- Python 3.11 + FastAPI
- Docker, приватный Registry
- Kubernetes (Yandex Cloud), Helm
- Jenkins (CI), ArgoCD (CD)
- Prometheus + Grafana (метрики), Loki (логи)
- Nginx Ingress, cert-manager (HTTPS)

## Безопасность и обслуживание

Секреты не хранятся в Git в открытом виде — используется Sealed Secrets. Данные в etcd зашифрованы на уровне хранилища (Encryption at Rest). Бэкапы etcd выполняются по расписанию, ключи шифрования хранятся изолированно. Роллбэки выполняются через ArgoCD или откатом коммитов в репозитории манифестов.
